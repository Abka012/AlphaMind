# Agent Session - AlphaMind HFT Optimization

## Date
2024-04-17

## Goal
Improve Sharpe ratio and enable high-frequency trading (HFT) for forex trading bot

## Requirements from User
- Sub-second execution capability
- 1 pip typical broker spread target
- $1M capital base
- Focus on Sharpe ratio improvement

## Changes Made

### 1. Reduced Prediction Horizon
- Changed from 1000 ticks (~3-15 min) to 50 ticks (~10-30 sec)
- File: `train.py`, `backtest.py`, `main.py`
- Enables faster trade signals

### 2. Added Micro-Structure Features (9 new)
- `tick_direction_imbalance` - Tracks bid/ask pressure
- `spread_compression` - Low spread detection
- `tick_acceleration` - Second-order momentum
- `vwap_deviation` - Fair value deviation
- `volume_weighted_imbalance` - Order flow
- `consecutive_bid/ask` - Mini-trend detection
- `tick_position_normalized` - Normalized position

### 3. Tightened Trade Parameters
- SL: 10 → 5 pips
- TP: 20 → 10 pips
- Check interval: 15 → 3 seconds
- Added confidence-weighted position sizing

### 4. Added Session Filtering
- Trade only during active sessions (8am-8pm)
- Filter on spread conditions

### 5. Updated .gitignore
- Added AGENTS.md to avoid committing agent config

### 6. Training Optimizations
- Added MAX_SAMPLES = 500000 limit for faster training
- Sample last N samples (recent data)

## Files Changed
- `train.py` - Horizon, features, sampling
- `backtest.py` - Horizon, thresholds, SL/TP
- `main.py` - Thresholds, SL/TP, features, cycle time
- `indicators.py` - New HFT features
- `.gitignore` - Added AGENTS.md
- `README.md` - Updated documentation

## Commands Run
```bash
# Train models (with sampling for speed)
python train.py

# Backtest
python backtest.py
```

## Key Findings
- Direction prediction at tick level is ~50% (random) - difficult to beat
- HORIZON=25 too short, HORIZON=50 better balance
- Session/spread filtering helps quality
- Micro-structure features add signal but cannot overcome market noise
